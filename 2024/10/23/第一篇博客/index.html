第一篇技术Blog —基于Hexo搭建博客的记录

缘由:

由于n年之前就想搭建一个博客用来记录一下我的一些技术学习上遇到的问题 (踩过的坑),以及解决方案，在搭建这个博客之前，我就已经付诸行动，尝试搭建过基于WordPress的博客，奈何那时的钱包不够鼓 （现在也是）。

依稀记得当时租的腾讯云学生机，配置是1C1G，带宽只有可怜的 1 Mbps。结果就是加载等到花儿都谢了，遂放弃了用WordPress搭建博客的想法。后面我还尝试过用discuz搭建过论坛，但是由于不可抗力因素(服务器到期了)而被迫终止计划。

转眼间来到2024，在一次偶然间我了解到了一些其它的博客系统，像是Hexo、Hugo、Typecho等。决定再次搭建一个博客记录一些日常生活和学习的内容。说干就干，在挑了几天的服务器、反复权衡利弊后，我决定。。。。

白嫖Github！！！使用Hexo部署在GithubPage的方案。刚好我还有一个域名还没有到期。选这个方案的理由很简单—便宜且完全够用。一年可以省下100来块 (不排除后面会迁移到服务器上)。

经过了半个小时的资料搜集，便正式开始了搭建Hexo。首先去node.js官网下载了最新的安装包，和下载Git。

## 部署 Hexo 到 GitHub Pages
1. 修改 _config.yml 文件末尾的 Deployment 部分

修改完成后，执行hexo d推送到 Github。如果没有报错，这时访问我们的 GitHub 域名 https://用户名.github.io 就可以看到刚刚搭建好的博客了。
2. 绑定域名
+ 添加一个CNAME类型的解析，自定义域名前缀(如xxx)，设置目标值为用户名.github.io。
+ 在博客根目录创建一个名为CNAME的文件，在里面输入博客的解析地址。例如xxx.你的域名.com
+ 用CF代理的域名记得把转发取消掉，不然github生成不了https证书。
+ 打开仓库的 Settings -> Pages -> 在 Custom domain 输入博客的解析地址，然后 Save,等待一会儿把 EnforceHTTPS 勾选上。
+ 清除缓存等文件并重新发布网站

这样就齐活了。

接下来是踩到的一些坑

  • 后面我部署好安装主题的时候，一切都非常顺利。从安装到线下测试，但是部署到线上，傻眼了，不是404就是空白页。期间经过10多次的删除仓库重新部署，还是不行。。。
  • 我一度怀疑是node.js的问题，重装了两次nodejs，问题依旧。这时我陷入了深深的自我怀疑。
  • 突然，我发现了push上去的文件都是0kb，然后我去度娘找，怀疑是nodejs版本的问题。然后去hexo官网查看nodejs相对应的支持版本，连降6个大版本，问题依旧。
  • 最后 我决定重新部署一遍Hexo和安装近几年还在正常维护更新的主题，结果。。成了 = =
  • 从早上8点搞到下午3点，服了。
